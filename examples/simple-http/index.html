    Simple HTTP Tests

    This example demonstrates basic HTTP endpoint testing with request chaining, variable saving, and assertions. It simulates a user management API workflow.

    Test Specification

    name: "Request Chaining Test Suite"
description: "A test suite demonstrating request chaining with the test server"
version: "v1.0.0"
tests:
  - name: "Create and Chain User Operations"
    steps:
      - name: "Create first user"
        plugin: "http"
        config:
          method: "POST"
          url: "http://localhost:8080/users"
          headers:
            Content-Type: "application/json"
          body: |
            {
              "name": "John Doe",
              "email": "john@example.com",
              "role": "admin"
            }
        assertions:
          - type: "status_code"
            expected: 200
          - type: "json_path"
            path: ".name"
            expected: "John Doe"
          - type: "json_path"
            path: ".email"
            expected: "john@example.com"
          - type: "header"
            name: "Content-Type"
            expected: "application/json"
        save:
          - json_path: ".id"
            as: "first_user_id"
          - json_path: ".email"
            as: "first_user_email"

      # ... More steps in the full example ...

    Key Features Demonstrated

    1. HTTP Methods: The example shows various HTTP methods:

    2. POST: Creating users

    3. GET: Retrieving user information
    4. PUT: Updating user data

    5. DELETE: Removing users

    6. Assertions:

    7. Status code validation

    8. JSON response validation using JSONPath

    9. Header validation

    10. Variable Management:

    11. Saving response values using save

    12. Using saved variables in subsequent requests with {{ variable_name }}

    13. Request Chaining:

    14. Creating dependent requests
    15. Verifying changes from previous operations

    Running the Example

    1. Start the test server (included in the repository):
    go run for-contributors/test-server/main.go
    1. Run the test suite:
    rocketship run -af examples/simple-http/rocketship.yaml
